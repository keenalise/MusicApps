  → Switches back to "Workout"

QUICK START (3 MINUTES)
════════════════════════════════════════════════════════════════

1. Add Sessions to Navigation (MainActivity.kt):

   composable(Screen.Sessions.route) {
       val sessionViewModel: SessionManagementViewModel = viewModel()
       SessionsScreen(
           viewModel = sessionViewModel,
           onSessionSelected = { sessionId ->
               println("Session: $sessionId")
           }
       )
   }

2. Add Sessions Tab to BottomNav:

   BottomNavigationItem(
       selected = isSessionsSelected,
       onClick = { navController.navigate(Screen.Sessions.route) },
       icon = { Icon(Icons.Default.PlaylistPlay, ...) },
       label = { Text("Sessions") }
   )

3. Build & Run:

   ./gradlew clean build
   ./gradlew installDebugApk

That's it! Users now have session management! 🎉

DATABASE SCHEMA
════════════════════════════════════════════════════════════════

playback_sessions: 9 columns
  id, title, created_at, updated_at, current_index,
  last_position_ms, playback_state, repeat_mode,
  shuffle_mode_enabled, is_active

queue_items: 9 columns
  id, session_id (FK), media_id, uri, title, artist,
  album, duration_ms, position_in_queue, extras_json

Foreign Key: queue_items.session_id → playback_sessions.id
Cascade Delete: YES (orphans prevented)

API QUICK REFERENCE
════════════════════════════════════════════════════════════════

Getting ViewModel:
  val vm = viewModel<SessionManagementViewModel>()

Creating Session:
  vm.createNewSession("Playlist Name")

Switching Session:
  vm.switchToSession(sessionId, autoPlay = true)

Observing Sessions:
  val sessions = vm.allSessions.collectAsStateWithLifecycle()
  val active = vm.activeSession.collectAsStateWithLifecycle()

Managing Sessions:
  vm.renameSession(id, "New Name")
  vm.deleteSession(id)
